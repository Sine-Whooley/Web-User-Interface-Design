Question 2
HTTP:
HTTP is the foundation of data communication on the web. It defines the rules for requesting & transferring web resources, such as HTML files, Stylesheets and Images.
HTTPS:
HTTP is the version of HTTP. It uses encryption (SSL/TLS) to protect the data exchanged between the client and the server. This ensures the integrity and confidentiality of Data.
TCP / IP Protocols:
  • When data is sent over the Internet, a protocol or agreed specifcation called the 'Transmission Control Protocol Internet Protocol' is used.
  • TCP takes care of the communication between your software application & your network software.
  • Its responsible for breaking data down into IP packets.
  • IP is responsible for 'rooting' each packet to its destination.
  • Its responsible for sending the packets to the reciever.
FTP:
  • File Transfer Protocol
  • It enables the transfer of files such as your web files from your local machine, on to a web server.

URL:
  • URL's are web addresses that identify resources on the iternet.
  • They consist of Protocols (eg. HTTP), Domain name's (eg. www.examplewebsite.com), and Path (eg. /page.html)
DNS:
  • DNS translates human-readable domain names info IP addresses, allowing browsers to locate web servers by name rather than numerical IP addresses
